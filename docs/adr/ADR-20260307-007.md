# ADR-20260307-007: cocoindex proxy モードのデフォルト設定維持（stdio）

- **ステータス**: accepted
- **日付**: 2026-03-07
- **決定者**: yoshihiko
- **関連**: cocoindex パッケージ, ADR-20260302-006

## コンテキスト

ADR-006 で mcp-proxy による HTTP 共有化（v2）を導入し、`proxy.enabled: false` をデフォルトとした。実運用では複数の Claude Code セッションが同一プロジェクトで並列起動するのが基本であり、proxy モードの方が SQLite ロック競合を回避できるため、デフォルトを `true` に変更すべきか検討した。

## 検討した選択肢

### 選択肢 A: デフォルトを `proxy.enabled: true` に変更

- メリット: 並列セッション時の SQLite 競合を配布先でもデフォルトで回避できる
- デメリット: 初回セッションで proxy 起動が MCP 接続に間に合わず `/mcp` で手動リコネクトが必要になる
- デメリット: 配布先に `mcp-proxy` がインストールされている前提になる
- デメリット: 初回体験の悪化（stdio なら Claude Code が直接起動するため問題なし）

### 選択肢 B: デフォルトを `proxy.enabled: false` のまま維持

- メリット: 配布先で追加依存なしに動作する
- メリット: 初回セッションでも MCP 接続が自動的に成功する
- デメリット: 並列セッション時の SQLite 競合は各プロジェクトで `local.yaml` を設定して回避する必要がある

## 決定

**選択肢 B: デフォルトを `proxy.enabled: false` のまま維持**する。

理由:

1. **配布先の初回体験を優先**: proxy モードは初回セッションで手動リコネクトが必須であり、新規プロジェクトへの配布時に混乱を招く
2. **追加依存の回避**: `mcp-proxy` のインストールを全配布先に強制するのは負担が大きい
3. **オプトインで十分**: 並列セッションが必要なプロジェクトでは `cocoindex.local.yaml` に `proxy.enabled: true` を追加すれば切り替わる（SessionStart hook が `.mcp.json` を自動書き換え）

## 影響

- `cocoindex.yaml` のデフォルト値は変更なし（`proxy.enabled: false`）
- 並列セッションを多用するプロジェクトでは引き続き `local.yaml` でのオプトインが必要
- ADR-006 の設計方針（v1 を壊さない、オプトイン方式）と一貫性を維持
