# ADR-20260423-018: cocoindex proxy 停止は supervisor の idle shutdown で扱う

- **ステータス**: accepted
- **日付**: 2026-04-23
- **決定者**: AI Orchestra maintainers
- **関連**: cocoindex パッケージ, ADR-20260302-006, ADR-20260421-017

## コンテキスト

ADR-20260421-017 で `proxy.enabled: true` を proxy-only とし、未 ready の初回 session は reconnect で扱う方針を採用した。
ただし停止については保留しており、次の問題が残っていた。

- `SessionEnd` hook は Claude Code の session 終了しか直接観測できない
- Codex / Gemini 側が最後に落ちた場合、Claude 側 hook だけでは「全 client がいなくなった」を判断できない
- stock `mcp-proxy` には idle shutdown がなく、`api_last_activity` だけでは静かな接続と本当の無接続を安全に区別しにくい

このため、停止責務を Claude hook に持たせず、proxy の入口側で client 接続を直接観測する仕組みが必要になった。

## 検討した選択肢

### 選択肢 A: `SessionEnd` と session registry で停止する

- メリット:
  - 実装差分が小さい
- デメリット:
  - Codex / Gemini の終了を確実に拾えない
  - stop 条件の正確さが Claude hook に依存する

### 選択肢 B: `mcp-proxy` の `/status` だけで idle 停止する

- メリット:
  - wrapper を増やさずに済む
- デメリット:
  - stock `mcp-proxy` には idle shutdown 機能がない
  - `api_last_activity` だけでは接続中だが静かな session を誤停止しうる

### 選択肢 C: 外側に supervisor を置き、接続数を見て idle 停止する

- メリット:
  - Claude / Codex / Gemini を横断して同じ観測点で停止判定できる
  - 外向き URL を変えずに導入できる
  - `start/stop/status` を既存の proxy 管理コマンドに乗せやすい
- デメリット:
  - transport 層に 1 レイヤー追加される
  - forwarding の互換性を重点テストする必要がある

## 決定

選択肢 C を採用する。

- `proxy_supervisor.py` を追加し、外側の固定ポートを supervisor が listen する
- 実際の `mcp-proxy` は inner の一時ポートで起動し、supervisor が TCP 転送する
- `proxy_manager.start_proxy()` / `stop_proxy()` / `status` は supervisor を管理対象にする
- `SessionEnd` は stop の契機に使わず、session state cleanup のみを行う
- supervisor は TCP 接続数ベースで `active_clients` を追跡する
- `active_clients == 0` が `idle_timeout` 秒続いたら、supervisor が inner proxy と自分自身を停止する
- `idle_timeout` は `cocoindex.yaml` の `proxy.idle_timeout` で管理する

## 影響

- 外向き URL は維持される
  - Claude Code / Gemini CLI: `/sse`
  - Codex CLI: `/mcp`
- `proxy_state` は `starting` / `ready` / `idle` / `stopping` / `stopped` / `failed` を取る
- `idle` は「即再利用可能だが active client は 0」の状態として扱う
- `notify-proxy-reconnect.py` と `provision-mcp-servers.py` は `idle` を `ready` と同等に扱う
- `orchex proxy stop/status` は supervisor ベースの状態表示に変わる
