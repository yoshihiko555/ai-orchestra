# ADR-20260414-015: agent-routing の未分類リサーチ入力は researcher 基点で解決する

- **ステータス**: accepted
- **日付**: 2026-04-14
- **決定者**: yoshihiko
- **関連**: ADR-20260223-005

## コンテキスト

`agent-routing` には、通常のエージェントトリガーに一致しない入力に対して、
PDF・動画・画像・コードベース全体の解析を Gemini CLI に寄せる
`GEMINI_FALLBACK_TRIGGERS` が存在していた。

しかしこの設計には 2 つの問題があった。

- `agent-routing` の責務は「どのエージェントへ寄せるか」の判断であり、
  実行ツールを Gemini に固定するのは `cli-tools.yaml` を SSOT とする
  config-driven 方針（ADR-20260223-005）とずれる
- `prompt.lower()` と `"PDF見て"` の比較不整合により、
  `このPDF見てください` が無反応になる実装バグがあった

## 検討した選択肢

### 選択肢 A: Gemini 固定フォールバックを維持する

- メリット: 実装変更が最小
- デメリット: `researcher.tool` を変更しても追従できず、config-driven 原則に反する
- デメリット: 未分類リサーチ入力だけ実行ツールがハードコードされる

### 選択肢 B: 未分類入力を researcher に寄せ、tool は config から解決する

- メリット: `agent-routing` の責務をエージェント選択に限定できる
- メリット: `researcher.tool` が `gemini` / `codex` / `claude-direct` のどれでも追従できる
- メリット: 通常ルーティングと同じ `[Agent Routing]` 出力形式に揃えられる
- デメリット: フォールバック処理の文言とテスト更新が必要

### 選択肢 C: 未分類リサーチ入力のフォールバック自体を廃止する

- メリット: ルーティング規則が単純になる
- デメリット: PDF/動画/画像/コードベース全体の解析依頼を拾えなくなる

## 決定

**選択肢 B** を採用。

具体的な決定:

1. `GEMINI_FALLBACK_TRIGGERS` は廃止し、`researcher` 向けのフォールバックトリガーとして扱う
2. フォールバック時も `researcher` をエージェントとして選び、
   実行 tool は `get_agent_tool("researcher", config)` で解決する
3. フォールバック出力も通常ルーティングと同じ `[Agent Routing]` 形式に揃える
4. 比較時はトリガー側も小文字化し、`PDF見て` の取りこぼしを防ぐ

## 影響

- `agent-routing` が config-driven 原則により忠実になる
- `researcher.tool` の将来的な変更が未分類リサーチ入力にも自動反映される
- `このPDF見てください` のような入力が期待どおり `researcher` に寄る
