# ADR-20260322-011: facet build 完全委譲（rules 廃止・knowledge/scripts 層導入）

- **ステータス**: accepted
- **日付**: 2026-03-22
- **決定者**: yoshihiko
- **関連**: Issue #23, ADR-20260322-010 (manifest-SSOT)

## コンテキスト

ADR-010 で `packages/*/skills/` を廃止し facet build に委譲したが、rules は二重管理のまま残っていた。

- `packages/*/rules/*.md` に rules のソースファイルが存在
- `facets/instructions/*.md` + `facets/compositions/*.yaml` にも同じ内容が存在
- sync エンジンが packages から rules をコピーし、facet build も rules を生成 → 二重配布
- `remove_stale_files` が facet build で生成したファイルを stale 扱いで誤削除するリスク

さらに、facet composition で knowledge（参考資料）や scripts を宣言的に管理する仕組みがなかった。

- スキルが参照する知識ファイルやスクリプトが `packages/*/skills/` に散在していた（Issue #20 で削除済み）
- facet build の出力に references/ や scripts/ を含める手段がなかった

## 検討した選択肢

### 選択肢 A: rules も facet build に完全委譲 + knowledge/scripts 層を導入（採用）

- メリット: skills と rules で一貫した管理方式（facet build が唯一の生成元）
- メリット: sync エンジンの責務が明確（agents/config のみ）
- メリット: composition.yaml に knowledge/scripts を宣言するだけでリソースが自動配布
- デメリット: 既存プロジェクトで synced_files に rules が残っている場合の移行対応が必要

### 選択肢 B: rules は packages に残し sync で配布を継続

- メリット: 既存の仕組みを変更しない
- デメリット: skills と rules で管理方式が異なり一貫性がない
- デメリット: facets/instructions と packages/rules の二重管理が永続する

## 決定

**選択肢 A を採用**。

具体的な変更:

### rules 廃止
1. `packages/*/rules/` ディレクトリを全削除（5パッケージ12ファイル）
2. `sync_packages()` から rules カテゴリを除外（agents/config のみ同期）
3. `orchestra-manager.py` の uninstall から skills/rules 削除ロジックを除去
4. `remove_stale_files` に `facet_managed` 除外を追加（移行時の誤削除防止）

### knowledge/scripts 層導入 (Issue #23)
5. `facets/knowledge/` に知識ファイルを集約（14ファイル）
6. `facets/scripts/` にスクリプトファイルを集約（1ファイル）
7. `composition.yaml` に `knowledge` / `scripts` フィールドを追加
8. FacetBuilder が `references/` と `scripts/` を自動配布

### 不要ファイル削除
9. `facets/evals/` を削除（配布対象外・未参照の proof of concept）

## 影響

### リソース管理の責務分離

| リソース | 管理元 | 配布方法 |
|---------|--------|---------|
| agents | `packages/*/agents/` | sync（mtime ベースコピー） |
| config | `packages/*/config/` | sync（mtime ベースコピー） |
| hooks | `packages/*/hooks/` | 直接参照（`$AI_ORCHESTRA_DIR`） |
| skills | `facets/` | facet build で生成 |
| rules | `facets/` | facet build で生成 |
| knowledge | `facets/knowledge/` | facet build で `references/` に配布 |
| scripts | `facets/scripts/` | facet build で `scripts/` に配布 |

### 移行時の注意

- 既存プロジェクトの `orchestra.json` の `synced_files` に rules パスが残っている場合、`remove_stale_files` が facet build で生成した rules を誤削除しうる → `facet_managed` 除外で対策済み
- evals は配布対象外。ai-orchestra リポジトリ内でのスキル開発・チューニングに使用
