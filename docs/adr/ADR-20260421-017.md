# ADR-20260421-017: cocoindex proxy 起動は proxy-only とし、state file と reconnect 通知で扱う

- **ステータス**: accepted
- **日付**: 2026-04-21
- **決定者**: AI Orchestra maintainers
- **関連**: cocoindex パッケージ, ADR-20260302-006, ADR-20260307-007

## コンテキスト

`cocoindex` の proxy モードは、SQLite 競合を避けるために `mcp-proxy` で単一プロセス化する設計として導入された。
しかし実運用では、proxy 未起動の session 開始時に次の問題があった。

- `SessionStart` で proxy を起動しても、初回の MCP 接続には間に合わない
- `InstructionsLoaded` に寄せても current session の接続を救済できなかった
- 起動失敗時に stdio へ戻すと、proxy と stdio が混在しうる
- `project_dir` 由来の URL/port と runtime 状態の責務が混ざっていた

2026-04-21 に Claude Code `v2.1.116` の `--print` 実測で、少なくとも次の順序を確認した。

1. MCP 設定読み込み / 接続試行
2. `SessionStart`
3. `InstructionsLoaded`

この順序では、hook だけで current session の proxy 接続成功を保証できない。

## 検討した選択肢

### 選択肢 A: proxy 起動失敗時は stdio fallback する

- メリット:
  - current session が使えなくなる可能性を減らせる
- デメリット:
  - proxy と stdio の mixed mode が発生しうる
  - SQLite 競合回避という proxy 導入目的と衝突する
  - session ごとに transport がぶれて挙動が読みにくい

### 選択肢 B: `proxy.enabled: true` を proxy-only とし、初回失敗は reconnect で扱う

- メリット:
  - transport の意味が一貫する
  - URL を deterministic に provision できる
  - startup と provision の責務を分離しやすい
- デメリット:
  - proxy 未起動の初回 session は手動 reconnect が必要になりうる
  - runtime state 管理が追加で必要になる

## 決定

選択肢 B を採用する。

- `proxy.enabled: true` は **proxy-only** とする
- proxy 未 ready 時でも stdio fallback は行わない
- `get_proxy_config()` を host/port の SSOT とし、URL 組み立ても `proxy_manager.py` に集約する
- `SessionStart` は
  - 各 CLI 設定の reconcile
  - session state の作成
  - proxy 未 ready 時のバックグラウンド warmup
  を担う
- `UserPromptSubmit` で、`proxy_state == ready` になった後に 1 回だけ reconnect を案内する
- runtime state は設定ファイルに入れず、`.claude/state/` に保存する
  - global: `.claude/state/cocoindex-proxy.json`
  - session: `.claude/state/cocoindex-sessions/<session_id>.json`
- `SessionEnd` は session state cleanup のみを行い、proxy 自体は停止しない

## 影響

- proxy モードの failure mode が明確になる
  - 初回 session は失敗しうる
  - warmup 完了後に `/mcp` reconnect で復旧する
- `provision-mcp-servers.py` は「起動結果に応じて transport を変える」責務を持たなくなる
- `notify-proxy-reconnect.py` と `start-mcp-proxy.py` が新たに追加される
- `cocoindex-usage.md` などの docs は、旧来の `InstructionsLoaded` 前提ではなく実測順序に合わせて更新する
- proxy の自動停止は別論点として残し、将来的に proxy 側 supervisor / idle shutdown を検討する
