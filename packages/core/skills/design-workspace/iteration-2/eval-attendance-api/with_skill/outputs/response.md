## 設計の進め方（確認後に実行するプラン）

上記の確認が取れたら、以下の順序で設計を進めます：

### Step 1: アーキテクチャ設計
- システム構成（API サーバー単体 or マイクロサービス等）
- 技術スタック確定
- 出力: `docs/architecture/architecture.md`

### Step 2: API 設計（一覧）
主要エンドポイントの一覧を設計します：
- `POST /attendance/clock-in` — 出勤打刻
- `POST /attendance/clock-out` — 退勤打刻
- `GET /attendance` — 勤怠一覧取得（フィルタリング・ページング方針含む）
- `GET /reports/monthly/{year}/{month}` — 月次レポート生成
- 認証エンドポイント
- 出力: `docs/api/api-list.md`

### Step 3: データモデル設計
主要エンティティ（従業員、勤怠記録、月次集計等）のリレーションとテーブル定義を整理します。
- 出力: `docs/database/er-design.md`
