# Design スキル 初回応答

## フェーズ判定

- ユーザーの意図: 「要件定義から始めたい」→ **Phase 1 から開始**
- 既存システムへの機能追加と明示 → **Phase 0（既存システム調査）を先行実施**

## 進め方の説明

ECサイトへのレビュー機能追加、了解しました。

「既存のECサイトへの機能追加」とのことなので、まず **Phase 0（既存システム調査）** として現在のシステムの技術スタックや構成を把握し、その上で要件定義を進めます。

### 全体の進め方

```
Phase 0: 既存システム調査  ← まずここから
    ↓
Phase 1: 要件定義（何を作るか）
    ↓
Phase 2: 基本設計（どう作るか）
    ↓
Phase 3: 詳細設計（実装仕様）
```

既存システムのコードベースがこのプロジェクト内にある場合は調査しますが、まず現状の把握のために確認させてください。

---

## AskUserQuestion ツール呼び出し（初回）

```
AskUserQuestion(
  "既存ECサイトの技術スタックについて教えてください。

  1. フロントエンド・バックエンドの言語・フレームワーク（例: Next.js + Django、Rails、Laravel など）と、データベース（例: PostgreSQL, MySQL）は何を使っていますか？
  2. 現在のコードベースはこのプロジェクト配下にありますか？それとも別リポジトリ・別環境ですか？（コードを直接調査できるか確認したいです）"
)
```

---

## 補足: 次ターン以降の質問予定

Phase 0 の回答を得た後、Phase 1（要件定義）の入り口として以下を確認します。

**次の質問（2ターン目予定）:**
- レビュー機能のスコープ（商品購入者のみ投稿可能か、誰でも投稿可能か）
- 星評価のスケール（5段階か、10段階か）

**その後（3ターン目以降）:**
- 非機能要件（パフォーマンス、モデレーション機能の有無、スパム対策）
- スコープ外の確認（レビューへの返信機能、画像添付など）

---

## 用語整理（初期案）

Phase 1 で整理予定の主要用語:

| 用語 | 定義（案） |
|------|----------|
| レビュー | 商品に対してユーザーが投稿する評価（星評価 + テキスト）|
| 星評価 | 数値による評価スコア（スケール未定） |
| レビュアー | レビューを投稿するユーザー |
| 評価集計 | 商品に紐づく全レビューの平均評価・件数 |

（実際の定義はユーザーとの対話で確定させる）
