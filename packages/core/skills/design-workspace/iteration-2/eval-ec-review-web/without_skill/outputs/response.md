# ECサイト レビュー機能 — 要件定義・設計

## 概要

既存ECサイトへのレビュー機能追加について、要件定義から設計の方針を整理します。

---

## 1. 要件定義

### 機能要件

#### 投稿側（ユーザー）

- 購入済み商品に対して星評価（1〜5）とテキストを投稿できる
- 1ユーザー × 1商品につき1件のみ投稿可能（重複投稿禁止）
- 投稿したレビューを編集・削除できる
- レビュー投稿はログイン済みユーザーのみ可能

#### 閲覧側（全ユーザー）

- 商品詳細ページにレビュー一覧を表示する
- 星評価の平均と件数を表示する
- レビューを新着順・評価順で並べ替えできる

#### 管理側（管理者）

- 不適切なレビューを非表示・削除できる
- レビュー一覧を管理画面から確認できる

### 非機能要件

- レビュー一覧は商品詳細ページのロードに影響しない（遅延ロードまたはページネーション）
- スパム・不適切コンテンツへの対策（将来的にフィルタリング機能を拡張できる設計）
- 既存の認証・ユーザー管理との統合

### スコープ外（今回は含めない）

- 画像付きレビュー
- 他ユーザーのレビューへの「参考になった」ボタン
- レビュー通知機能
- AI による自動モデレーション

## 2. データ設計

### エンティティ: `reviews` テーブル

| カラム名 | 型 | 説明 |
|---|---|---|
| `id` | BIGINT PK | レビューID |
| `product_id` | BIGINT FK | 対象商品ID |
| `user_id` | BIGINT FK | 投稿ユーザーID |
| `rating` | TINYINT | 星評価（1〜5） |
| `title` | VARCHAR(100) | レビュータイトル（任意） |
| `body` | TEXT | レビュー本文 |
| `status` | ENUM | `published` / `hidden` |
| `created_at` | TIMESTAMP | 投稿日時 |
| `updated_at` | TIMESTAMP | 更新日時 |

**制約:**

- `(product_id, user_id)` にユニーク制約（1ユーザー1商品1件）
- `rating` は 1〜5 の CHECK 制約

**インデックス:**

- `product_id` + `status` + `created_at`（一覧取得用）
- `product_id` + `rating`（評価集計用）

### 商品テーブルへの追加（非正規化キャッシュ）

商品詳細ページで毎回集計クエリを走らせないよう、`products` テーブルに以下を追加する:

| カラム名 | 型 | 説明 |
|---|---|---|
| `review_count` | INT | レビュー件数 |
| `review_rating_avg` | DECIMAL(3,2) | 平均評価 |

レビュー投稿・編集・削除時にトリガーまたはアプリケーション層で更新する。

## 3. API 設計

### エンドポイント一覧

| メソッド | パス | 説明 | 認証 |
|---|---|---|---|
| GET | `/api/products/:id/reviews` | レビュー一覧取得 | 不要 |
| POST | `/api/products/:id/reviews` | レビュー投稿 | 必要 |
| PUT | `/api/reviews/:id` | レビュー編集 | 必要（本人のみ） |
| DELETE | `/api/reviews/:id` | レビュー削除 | 必要（本人 or 管理者） |
| PATCH | `/api/admin/reviews/:id/hide` | 管理者：非表示 | 管理者のみ |

### GET `/api/products/:id/reviews` — レスポンス例

```json
{
  "summary": {
    "count": 42,
    "average": 4.3,
    "distribution": { "5": 20, "4": 10, "3": 8, "2": 2, "1": 2 }
  },
  "reviews": [
    {
      "id": 101,
      "rating": 5,
      "title": "とても良かった",
      "body": "品質が高く満足しています。",
      "user_name": "田中 太郎",
      "created_at": "2026-03-01T10:00:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 42
  }
}
```

### POST `/api/products/:id/reviews` — リクエスト例

```json
{
  "rating": 5,
  "title": "とても良かった",
  "body": "品質が高く満足しています。"
}
```

**バリデーション:**

- `rating`: 必須、整数、1〜5
- `body`: 必須、1〜2000文字
- `title`: 任意、100文字以内

## 4. フロントエンド設計

### 商品詳細ページへの追加コンポーネント

```
ProductDetailPage
├── ProductInfo（既存）
├── ReviewSummary         ← 新規：平均評価・件数・評価分布バー
├── ReviewForm            ← 新規：ログイン済みかつ未投稿の場合に表示
└── ReviewList            ← 新規：レビュー一覧（ページネーション付き）
    └── ReviewItem × N
```

### UI 上の考慮点

- レビューフォームは購入済みユーザーにのみ表示（または購入していない場合はメッセージ表示）
- 星評価の入力はクリックまたはホバーで選択できるスター UI
- レビュー一覧はスクロール位置を保ちながら並び替え切り替えが可能

---

## 5. 実装の進め方（推奨フェーズ）

### Phase 1: バックエンド基盤（1〜2週間）

1. `reviews` テーブルのマイグレーション
2. `products` テーブルへのキャッシュカラム追加
3. レビューの CRUD API 実装
4. バリデーション・権限チェック

### Phase 2: フロントエンド実装（1〜2週間）

1. `ReviewSummary` コンポーネント
2. `ReviewList` + `ReviewItem` コンポーネント
3. `ReviewForm` コンポーネント（投稿・編集）
4. 商品詳細ページへの組み込み

### Phase 3: 管理機能（0.5〜1週間）

1. 管理画面でのレビュー一覧・非表示機能
