## 設計方針メモ（内部）

ユーザーが明示した主要機能:

1. 出退勤の打刻
2. 勤怠一覧の取得
3. 月次レポート生成

これらをもとに、第1ターンで確認する情報:

- **技術スタック**: APIの言語・フレームワーク、DB種別 → エンドポイント命名規則・型定義に影響
- **ユーザーロール・認証方式**: エンドポイントの認可設計に直結

次ターンで確認予定の情報（1回の質問は2〜3項目まで）:

- 打刻データの構造（位置情報・デバイス情報の要否）
- 月次レポートの出力形式（PDF/CSV/JSONなど）
- スコープ（残業計算・申請ワークフローはIn/Outか）
