## フェーズ判定

タスクプロンプトの分析:

- 「基本設計の画面設計を進めたい」 → **Phase 2 から開始**（要件は把握済みと判断）
- 「20画面以上のWebアプリ」 → デザインシステムトラックのトリガー条件に合致（10画面以上）
- 「5人のフロントエンドエンジニアが並行開発」 → デザインシステムトラックのトリガー条件に合致（複数FEエンジニア）
- 「UIの統一ルールも決めておきたい」 → ユーザーが明示的にデザインシステム関連キーワードを使用

設計トラック判定:
- **デザインシステムトラックをただちに開始する**（提案ではなく直接開始。理由: ユーザーが「UIの統一ルールを決めたい」と明示 + 20画面以上 + 複数FE開発者）
- `references/design-system.md` を読み込み済み

## スキル実行内容（第1ターン）

### ユーザーへの提示内容

ありがとうございます。状況を整理します。

**確認できた前提:**
- 新規開発（既存システムなし → Phase 0 スキップ）
- 20画面以上のWebアプリ
- 5名のFEエンジニアが並行開発
- 基本設計フェーズ（Phase 2）から開始

**進め方の提案:**

「UIの統一ルールを決めたい」と明示いただいているため、**画面設計（screen-list.md）とデザインシステム設計を並行**して進めます。
20画面以上・複数人開発というスケールでは、デザインシステムを先に固めておくことが開発効率と品質に直結するためです。

**実施予定の設計トラック:**

| トラック | 成果物 |
|---------|--------|
| Phase 2: 基本設計 | architecture.md, screen-list.md, screen-transitions.md, api-list.md, er-design.md |
| 拡張: デザインシステム | principles.md, tokens.md, component-list.md |

まず技術スタックとデザイン方針の基礎情報を確認させてください。

### AskUserQuestion（第1問）

```
AskUserQuestion("""
基本設計を進めるにあたり、以下を教えてください。

1. フロントエンドの技術スタックはすでに決まっていますか？
   （例: React + TypeScript, Vue 3, Next.js 等。未定の場合は候補や制約があれば）

2. UIデザインの方向性として、参考にしたいサービスや既存のデザインガイドラインはありますか？

3. このポータルサイトの主なユーザーは誰で、主にどの端末・環境で使用しますか？
   （例: 社内PC（デスクトップメイン）、スマートフォン併用、特定ブラウザ制約など）
""")
```

## メタ情報（評価用）

### トリガー条件の判定ログ

| 条件 | 判定 | 根拠 |
|------|------|------|
| 画面数 10以上 | ✅ | 「20画面以上」と明示 |
| 複数FE開発者 | ✅ | 「5人のFEエンジニアが並行開発」と明示 |
| ユーザーが明示的キーワード使用 | ✅ | 「UIの統一ルールも決めておきたい」 |
| → デザインシステムトラック提案ではなく直接開始 | ✅ | SKILL.md の規則: 「ユーザーが各トラックに関連するキーワードを使った場合は、提案ではなく直接 reference を読み込んで開始する」 |

### スキルルール適用

| ルール | 適用状況 |
|--------|---------|
| AskUserQuestion を使用 | ✅ |
| 1回の質問は2-3項目まで | ✅（3項目） |
| Phase判定を明示 | ✅（Phase 2 から） |
| 拡張トラックをただちに開始 | ✅（design-system.md 読み込み済み） |

### 次ターンの予定

ユーザーの回答を受けて:
1. 技術スタック確定 → architecture.md のドラフト作成
2. デザイン方針確定 → principles.md の骨格作成
3. ユーザー属性・端末確定 → レスポンシブ方針を決定（デスクトップファースト vs モバイルファースト）
4. その後、画面一覧のヒアリングへ進む（screen-list.md）
