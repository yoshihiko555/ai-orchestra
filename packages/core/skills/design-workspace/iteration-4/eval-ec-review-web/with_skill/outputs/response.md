## Phase 0: 既存システム調査

ECサイトへのレビュー機能追加ですね。既存システムの状況を把握したいので、いくつか確認させてください。

```
AskUserQuestion("既存のECサイトについて教えてください。

1. 技術スタックを教えてください。
   - フロントエンド（例: React, Vue, Next.js, 素のHTML等）
   - バックエンド（例: Django, Rails, Laravel, Express等）
   - データベース（例: PostgreSQL, MySQL, MongoDB等）

2. 認証・ユーザー管理はすでに実装されていますか？
   - ログイン済みユーザーの識別は可能か（レビューを投稿するユーザーを特定する必要があるため）

3. 既存の設計ドキュメント（ER図、API仕様書等）はありますか？あれば後で参照します。")
```

## （ユーザーの回答を受け取った後の想定フロー）

Phase 0 完了後、以下の流れで Phase 1（要件定義）に移行します。

### Phase 1 で整理予定の内容

ユーザーの回答を踏まえた上で、以下を対話で詰めていきます。

1. **プロジェクト概要** — レビュー機能の目的・背景・ターゲットユーザー
2. **用語定義** — 「レビュー」「評価」「星評価」等の用語を整理
3. **機能一覧** — レビュー投稿・表示・管理に関わる機能をリストアップ
4. **機能要件** — 各機能のユースケース・受け入れ条件
5. **非機能要件** — パフォーマンス、スパム対策、セキュリティ等
6. **優先順位付け** — MoSCoW で分類
