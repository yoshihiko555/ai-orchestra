# ECサイト レビュー機能 — 要件定義・設計開始

ありがとうございます。要件定義を始めるにあたり、既存システムと要件の詳細を確認させてください。

---

## 確認したい事項

### 1. 既存システムの技術スタック

- フロントエンド: どのフレームワーク/ライブラリを使用していますか？（例: React, Vue, Next.js, 素のHTML等）
- バックエンド: 言語・フレームワークは何ですか？（例: Python/Django, Node.js/Express, Go, Rails等）
- データベース: 何を使用していますか？（例: PostgreSQL, MySQL, MongoDB等）
- 認証: ユーザー認証は既に実装されていますか？（レビュー投稿者の特定に関わります）

### 2. レビュー機能の詳細要件

以下の点について教えてください：

**評価・投稿について**
- 星評価は何段階にしますか？（一般的には5段階）
- 1ユーザーが同一商品に複数回レビューを投稿できますか？（1商品1レビュー制限？）
- 購入済みユーザーのみ投稿可能にしますか？それとも全ユーザーが投稿可能ですか？
- テキストレビューは必須ですか？それとも星評価のみでも投稿可能ですか？
- テキストの文字数制限はありますか？

**モデレーション・管理**
- 投稿されたレビューは即時公開ですか？それとも管理者の承認フローが必要ですか？
- 不適切なレビューの報告機能（通報ボタン等）は必要ですか？
- 管理画面でレビューを管理する機能は必要ですか？

**表示について**
- レビューの並び替えは必要ですか？（新しい順、評価が高い順等）
- ページネーションは必要ですか？
- 他のユーザーが「このレビューは参考になった」と評価できる機能（役立ち投票）は必要ですか？

### 3. 非機能要件

- 想定ユーザー規模・商品数はどの程度ですか？（パフォーマンス設計に影響します）
- 対応ブラウザ・デバイスの要件はありますか？（スマートフォン対応必須か等）
- セキュリティ面で特別な要件はありますか？（スパム対策、レート制限等）

---

## 現時点での基本方針案

上記の回答を踏まえて詳細化しますが、一般的なECサイトのレビュー機能として、以下の方針を前提に進める予定です：

| 項目 | 想定方針 |
|------|----------|
| 評価方式 | 5段階の星評価 |
| 制限 | 購入済みユーザー・1商品1レビュー |
| 公開 | 即時公開（管理者による後審査） |
| 認証 | 既存ログイン機能を活用 |

確認事項への回答をいただけると、要件定義書と設計（DB設計・API設計・画面設計）に進めます。
