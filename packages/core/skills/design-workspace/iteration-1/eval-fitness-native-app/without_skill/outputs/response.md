# フィットネス記録アプリ — 設計前ヒアリング

設計を始める前に、いくつか前提を確認させてください。
決まっていない項目はその旨を教えていただければ、こちらで候補を提案します。

---

## 確認事項

### 1. 開発体制・技術スタック

- **クロスプラットフォーム戦略**: React Native / Flutter / ネイティブ（Swift + Kotlin）のどれを希望しますか？
  - 1チームで iOS/Android を同時開発したい → React Native or Flutter を推奨
  - パフォーマンス重視 / センサー・ヘルスkit 連携が多い → ネイティブも選択肢
- **チーム構成**: モバイル開発経験者はいますか？（技術選定に影響します）

### 2. バックエンド・データ管理

- **データの保存先**: クラウド同期（複数デバイス対応）が必要ですか？それともオフライン（端末内のみ）で十分ですか？
- **認証**: ユーザーアカウント機能（メール/SNSログイン）は必要ですか？
- **バックエンド**: 既存の API / インフラはありますか？ゼロから構築しますか？

### 3. 機能スコープの優先度

以下の機能について、MVP（最初のリリース）に含めるものを教えてください。

| 機能 | MVP必須 / 後回し / 不要 |
|------|------------------------|
| ワークアウト記録（種目・セット・レップ・重量） | ? |
| カスタム種目の追加 | ? |
| 体重・体脂肪率のトラッキング | ? |
| 目標設定（体重・体脂肪率・ワークアウト頻度など） | ? |
| 進捗グラフ・ビジュアル表示 | ? |
| Apple Health / Google Fit 連携 | ? |
| プッシュ通知（リマインダー） | ? |
| ワークアウトプランのテンプレート | ? |
| ソーシャル機能（共有・フォロー） | ? |

### 4. ターゲットユーザー・UX方針

- **主なユーザー像**: 初心者向け（シンプル重視）か、上級者向け（詳細記録・分析重視）か
- **デザイン参考**: 参考にしたいアプリ（Strong、Hevy、MyFitnessPal 等）はありますか？

### 5. 非機能要件

- **オフライン動作**: ネットワーク不要でも使えるべきですか？
- **リリース目標**: いつまでにどのストア（App Store / Google Play）にリリースしたいですか？

---

上記が揃えば、以下を設計として提示できます。

- アーキテクチャ全体図（フロントエンド・バックエンド・データ層）
- 画面設計（主要フロー）
- データモデル（ワークアウト・体重記録・目標）
- 技術スタック推奨と選定理由
- フェーズ別開発ロードマップ

決まっていない項目が多い場合は、一般的なベストプラクティスに基づいて推奨案を提示しながら設計を進めることもできます。
