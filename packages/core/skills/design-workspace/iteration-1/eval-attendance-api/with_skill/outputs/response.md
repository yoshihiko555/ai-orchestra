## レスポンス本文

承知しました。要件はほぼ固まっているとのことで、**Phase 2（基本設計）**から進めます。

今回のシステムは「API サーバー + DB」が中心の構成になりそうですね。画面設計（Step 2）は今回スキップして、以下の順番で進めましょう：

1. アーキテクチャ設計（技術スタック・システム構成）
2. API 設計（エンドポイント一覧・認証方式・共通仕様）
3. データモデル設計（テーブル定義・リレーション）

まず **アーキテクチャの前提を確認** させてください。以下の 2 点だけ教えてください。

### 確認事項（1 / 2）

**Q1. 技術スタック・制約はありますか？**

例えば：
- バックエンド言語・フレームワーク（Python/FastAPI、Go/Echo、Node.js/Express など）
- DB（PostgreSQL、MySQL、MongoDB など）
- 既存システムとの連携有無（例：社内の人事システムと連携が必要）
- デプロイ先の想定（AWS、GCP、オンプレ等）

**Q2. 認証はどうしますか？**

- 社員が各自ログインする形式（JWT / セッション認証）
- 既存の社内 SSO・LDAP 連携
- API キーのみ（別システムから呼ばれる形）

---

技術制約がなければこちらで推奨案を出しますので、「とくになし、おまかせ」でも大丈夫です。

どちらかだけでも分かれば先に進められます。
