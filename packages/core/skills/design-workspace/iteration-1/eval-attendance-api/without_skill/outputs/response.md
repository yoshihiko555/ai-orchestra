# 勤怠管理システム API 設計 — 初回レスポンス

要件を把握しました。「出退勤の打刻・勤怠一覧取得・月次レポート生成」を中心に API 設計とデータモデル設計を進めます。

設計の精度を上げるため、いくつか確認させてください。

---

## 確認事項

### 1. ユーザー・組織構造

- 社員は単一組織に所属しますか？それとも部署・チーム階層がありますか？
- 承認フロー（上長が勤怠を承認する）は必要ですか？

### 2. 打刻ルール

- 打刻手段はどれですか？（例: Web ブラウザ / モバイルアプリ / 両方）
- 位置情報（GPS）や IPアドレス制限など、打刻時の付加情報は必要ですか？
- 打刻の修正・取り消しは誰が行えますか？（本人のみ / 管理者も可）

### 3. 勤務種別・シフト

- 勤務種別（通常出勤・在宅・休暇・残業など）を区別しますか？
- シフト管理（予定勤務時間の定義）は本システムのスコープに含まれますか？

### 4. 月次レポートの内容

- レポートに含める主な集計値を教えてください（例: 総労働時間、残業時間、有給消化日数など）
- レポートの出力形式は何ですか？（JSON レスポンス / PDF / CSV）

### 5. 認証・権限

- 既存の認証基盤（SSO / OAuth など）はありますか？それとも本システムで認証も持ちますか？
- 権限ロールの想定を教えてください（例: 一般社員 / 管理者 / 人事担当など）

---

上記が不明な場合でも、一般的な想定をおいて「まず叩き台となる設計を出してほしい」であれば、そのままお伝えください。標準的な仮定をおいて API 仕様とデータモデルのドラフトを作成します。
