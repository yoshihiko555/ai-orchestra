## Phase 3 の進め方（勤怠管理API）

### 対象スコープ

勤怠管理APIのため **画面詳細設計（SC-xxx）はスキップ**し、以下の2種類の設計書を作成します：

| 種類 | ファイル | 内容 |
|------|---------|------|
| API 詳細設計 | `docs/api/API-001.md`〜 | 各エンドポイントのリクエスト/レスポンス仕様 |
| テーブル詳細設計 | `docs/database/TBL-001.md`〜 | 各テーブルのカラム定義・インデックス・制約 |

### 想定される成果物の構成例

```
docs/
├── api/
│   ├── api-list.md          ← 完成済み（一覧）
│   ├── API-001.md           ← 打刻API（出勤・退勤）詳細
│   ├── API-002.md           ← 勤怠一覧取得API 詳細
│   ├── API-003.md           ← 月次レポート生成API 詳細
│   └── ...
└── database/
    ├── er-design.md          ← 完成済み（全体像）
    ├── TBL-001.md            ← attendances テーブル詳細
    ├── TBL-002.md            ← employees テーブル詳細
    └── ...
```

## AskUserQuestion

`api-list.md` と `er-design.md` の内容を確認したいのですが、ファイルの場所が見当たりませんでした。以下の2点を教えてください。

**1. 設計ドキュメントの場所または内容**

- ドキュメントが特定のディレクトリにある場合、パスを教えてください
- または `api-list.md` と `er-design.md` の内容を直接貼り付けていただけますか

これにより、既存のAPI IDやテーブル名に対応した正確な設計書（`API-001.md`, `TBL-001.md` 等）を作成できます。

**2. 詳細設計の優先順位**

`api-list.md` の全エンドポイントと `er-design.md` の全テーブルを一気に詳細化するか、優先度の高いものから段階的に進めるかを教えてください。

- 全て一括で詳細化する（一覧が少ない場合向き）
- Must 優先度のものから順に進める（一覧が多い場合向き）
